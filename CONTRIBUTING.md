# Contributing to Code Particles

We welcome contributions that reduce complexity, improve clarity, or strengthen guarantees across our projects.

## Before you start

* Read the relevant project `README` and governance notes.
* Align with our philosophy: simplicity, safe defaults, accessibility, performance, and longevity.
* Open an issue to discuss large changes before submitting code.

## How to contribute

1. Fork the repository you want to change.
2. Create a focused branch for your work.
3. Add tests or examples where they improve confidence.
4. Submit a pull request that explains the problem and the reasoning behind the solution.

## Review guidelines

* Favor removing complexity over adding knobs.
* Prefer boring, stable solutions to clever ones.
* Keep accessibility and performance as non-negotiable requirements.
* Document behavior and decisions in the PR description.

## Communication

* Use GitHub issues for proposals and questions.
* Keep discussions constructive, direct, and respectful.
